# Windows Process Cleaner

🇷🇺 Русский · [🇬🇧 English](README.en.md)

Утилита для Windows: находит и завершает **забытые / зависшие процессы** и очищает
**Standby Memory**. Работает как с процессами разработки (node, python, java, vite,
webpack, npm/pnpm/yarn…), так и — в глобальном режиме — с **любыми** осиротевшими
и простаивающими процессами.

Написана на **C# + WinForms**, собирается **встроенным в Windows компилятором
`csc.exe`** (.NET Framework 4.x). **Ничего устанавливать не нужно** — ни Node.js,
ни Rust, ни Visual Studio. Результат — самодостаточный `.exe`.

---

## Для чего эта программа

Один инструмент, чтобы навести порядок на машине разработчика: убрать зря висящие
процессы, освободить оперативную память и вычистить накопившийся мусор с диска — без
установки чего-либо и без сторонних «оптимизаторов». Всё в одном окне, с русским и
английским интерфейсом, с работой из трея и по расписанию.

## Что программа умеет

- **Процессы.** Находит забытые/заброшенные процессы (dev-рантаймы или — в глобальном
  режиме — любые ваши) и безопасно их завершает: сначала мягко (WM_CLOSE), потом
  принудительно. Показывает по каждому: PID, PPID, путь, аптайм, CPU, RAM, окна,
  TCP-порты, дочерние процессы, владельца.
- **Оперативная память.** Очищает Standby Memory через WinAPI (`NtSetSystemInformation`),
  без сторонних exe.
- **Dev Cleanup.** Массово завершает Node/Python/Java/Vite/Webpack/npm/pnpm/yarn и
  освобождает занятые dev-порты (3000, 5173, 8080, 4200 …).
- **Очистка диска.** Анализирует и удаляет известный мусор: dev-кэши (npm/pnpm/yarn/
  pip/gradle/cargo/go/NuGet), системный мусор (temp, Корзина, Windows Update, дампы),
  кэши браузеров и приложений (Discord/Slack/Teams/Spotify), старые логи, installer-мусор
  драйверов и `Windows.old`.
- **Docker.** Показывает занятое место (`docker system df`) и чистит неиспользуемое:
  остановленные контейнеры, образы, тома, кэш сборки, всё сразу.
- **Программы.** Список установленного ПО с удалением через штатный деинсталлятор.
- **Автозапуск.** Список всех установленных программ с тоглом «в автозапуске Windows»:
  включаешь — добавляет, выключаешь — убирает (реестр `Run` + папка «Автозагрузка»).
- **Автоматизация.** Таймер автоочистки процессов (каждые 1–24 ч), автозапуск с Windows
  (через Планировщик), работа в трее, история очисток, гибкие настройки.
- **Оформление.** Тёмная/светлая тема (или по системе) + тёмный заголовок окна;
  язык интерфейса RU/EN.

## Чего программа НЕ делает (границы безопасности)

- **Не ломает Windows.** Системные процессы (SYSTEM/сервисы), компоненты из `C:\Windows`,
  критичные и защищённые процессы (оболочка, облака, драйверы, мессенджеры) в глобальном
  режиме под завершение не попадают; предохранители работают «в сторону защиты».
- **Не завершает активное по ошибке.** Кандидат в скане отбирается только при совпадении
  всех критериев сразу (мёртвый родитель + простой + нет окон/портов/детей), с
  подтверждением. Исключение — кнопки Dev Cleanup: это осознанная «кувалда» по имени.
- **Не ищет дубликаты файлов по диску** и не удаляет ничего из ваших проектов, кода,
  `DriverStore`, System32 и корней дисков. Чистятся только известные мусорные пути.
- **Не чистит диск и не удаляет программы по расписанию** — только вручную, с
  предпросмотром и подтверждением. По таймеру работает лишь очистка процессов.
- **Docker: удаляет только неиспользуемое** (`prune`) — запущенные контейнеры и
  используемые образы не трогает. Kubernetes не включён.

## Быстрый старт (одна команда)

Дважды кликни по **`run.bat`** — при первом запуске он соберёт
`WindowsProcessCleaner.exe` и запустит его. Дальше можно запускать сам `.exe`.

Только сборка, без запуска:

```
build.bat
```

> ⚠️ Приложение запускается **от имени администратора** (нужно для очистки Standby
> Memory). При запуске появится запрос UAC — это нормально.

## Возможности

### Сканирование
По каждому процессу собирается: имя, PID, PPID, путь, время работы, CPU %, RAM,
наличие окна, слушающие TCP-порты, наличие дочерних процессов, владелец.

### Критерии «заброшенности»
Процесс становится кандидатом на завершение, только если выполнены **все** условия:
1. родительский процесс завершён;
2. CPU ниже порога дольше заданного времени простоя;
3. нет пользовательских окон;
4. не слушает TCP-порты;
5. нет дочерних процессов;
6. не в белом списке и старше минимального времени жизни.

### Два режима поиска
- **Dev-режим** (по умолчанию) — только процессы из списка отслеживаемых
  (node, python, java, vite, webpack, npm, pnpm, yarn, bun, cargo, go, deno, ruby, php…).
- **Глобальный режим** (галочка «Все процессы (глобально)») — сканируются **все**
  процессы системы. Здесь действуют **усиленные предохранители**: завершаются только
  процессы **текущего пользователя**, лежащие **не в системных папках** Windows и
  **не в `Program Files`** (установленный софт, настраивается), простаивающие **≥ 30
  минут** (настраивается), не входящие в расширенный список защищённых (ядро Windows,
  оболочка, облака, мессенджеры, драйверы, лаунчеры, менеджеры паролей). Именно так
  отлавливаются осиротевшие группы, у которых родитель давно умер, а дети остались
  висеть и грузить проц/память, ничем не используясь.

### Кнопки очистки
- **Очистить выбранные** — завершить отмеченные в таблице.
- **Автоочистка всех неактивных** — завершить всех найденных кандидатов одним
  нажатием (с подтверждением и списком).
- **Выбрать все / Снять выбор** — управление галочками.
- **Очистить память** — только очистка Standby Memory без завершения процессов.

Само завершение: сначала мягко (WM_CLOSE), ожидание до 3 секунд, затем принудительно.

### Dev Cleanup
Массовое завершение по группам: всё Node / Python / Java / Vite / Webpack / npm /
pnpm / yarn·bun / Docker Compose / Go·Cargo·Deno. Плюс список процессов, занявших
популярные dev-порты (3000, 5173, 8080, 4200 …) с возможностью их завершить.

> В Dev Cleanup завершение идёт **по имени, не глядя на активность** (кроме белого
> списка) — это осознанная «кувалда».

### Очистка диска (только вручную)
Отдельная вкладка. Флоу: **Анализировать → предпросмотр с размерами → удалить
выбранное**. Автоочистки диска нет намеренно (файлы менее обратимы, чем процессы).
Чистятся только **известные мусорные пути**, никакого поиска дубликатов по диску.
Категории:
- **Dev-кэши** — npm / pnpm / yarn / pip / gradle / cargo / go / NuGet (пересоздаются).
- **Системный мусор** — `%TEMP%`, `Windows\Temp`, Корзина, кэш Windows Update,
  crash dumps, отчёты об ошибках.
- **Кэши браузеров** — Chrome / Edge / Brave / Firefox, только кэш (без паролей и куки).
- **Кэши приложений** — Discord / Slack / Teams / Spotify, только кэш.
- **Старые логи** — логи CBS/DISM, npm/yarn, отчёты об установке.
- **Старые драйверы + Windows.old** — installer-мусор NVIDIA/AMD, папка старой Windows.

Предохранители: пропускаются занятые файлы и точки повторного разбора (junction),
никогда не трогаются `DriverStore`, System32, корни дисков, код и проекты.

### Программы (деинсталляция)
Вкладка со списком установленных программ (имя, версия, издатель, размер). Отмечаешь
и удаляешь через приложение — запускается штатный деинсталлятор программы.

### Автозапуск
Вкладка со списком всех установленных программ и чекбоксом-тоглом. Галочка стоит —
программа в автозапуске Windows; поставить — добавить, снять — убрать. Читает и
пишет автозапуск из реестра (`HKCU\...\Run`, `HKLM\...\Run`) и папок «Автозагрузка»
(пользователь + общая). Оранжевым помечаются записи автозапуска, которых нет в списке
установленных программ (скрипты, ярлыки) — их тоже можно отключить. Добавление всегда
идёт в пользовательский `HKCU\...\Run`; по умолчанию, если программа не в автозапуске,
тогл выключен.

### Docker
Вкладка для очистки Docker (нужен Docker CLI + запущенный демон). Кнопки: обзор
занятого места (`docker system df`), удалить остановленные контейнеры, неиспользуемые
образы, неиспользуемые тома, очистить кэш сборки, полная очистка всего неиспользуемого.
Удаляется только **неиспользуемое** (`prune`) — запущенные контейнеры и используемые
образы не трогаются.

> ⚠️ **Важно про место на диске.** Docker Desktop хранит всё в одном растущем
> виртуальном диске WSL2 (`docker_data.vhdx`). `prune` освобождает место **внутри**
> этого диска, но сам файл на Windows **не уменьшается**. Чтобы реально вернуть место
> на диске Windows — кнопка **«Сжать диск Docker»**: она останавливает Docker
> (`wsl --shutdown`) и сжимает vhdx через `diskpart compact vdisk`, показывая размер
> до/после. Все запущенные контейнеры при этом завершаются.

Kubernetes не включён (его очистка бьёт по живому кластеру).

### Язык интерфейса
Русский / English — переключается в настройках (применяется после перезапуска).
Документация тоже двуязычная: [README.md](README.md) / [README.en.md](README.en.md).

### Таймер автоочистки
Задаёшь число часов (1..24). Каждые N часов приложение само сканирует **процессы**
(в выбранном режиме), завершает кандидатов, чистит память и пишет в историю. Очистка
диска и деинсталляция по таймеру **не выполняются** — только вручную.

### Системный трей
Иконка в трее меняет цвет: зелёная — чисто, оранжевая — есть кандидаты. Двойной
клик — открыть окно. Правый клик — меню: Сканировать, Очистить, Очистить Standby
Memory, вкл/выкл автоочистку, перезапуск от администратора, выход. Закрытие окна
сворачивает приложение в трей (продолжает работать в фоне).

### Автозапуск с Windows
Галочка в настройках. Реализован через **Планировщик задач** с наивысшими правами
(`schtasks /RL HIGHEST`), поэтому при входе в систему UAC не появляется.

### Темы оформления
Светлая / тёмная / **по системе** (по умолчанию — следует за темой Windows).
Переключается в настройках, применяется сразу, включая заголовок окна.

### Настройки (сохраняются)
Порог CPU, время простоя, минимальное время жизни, период автоочистки, список
отслеживаемых процессов, белый список, dev-порты, тема, автозапуск, старт в трее,
глобальный режим.

### История
После каждой очистки сохраняются дата/время, количество завершённых процессов,
объём освобождённой памяти и список процессов.

## Безопасно ли это?

**Режим сканирования/автоочистки — да.** Кандидат отбирается только при совпадении
**всех** критериев сразу, поэтому активный процесс (с окном, слушающий порт, грузящий
CPU или с живым родителем) под удар не попадёт. В глобальном режиме дополнительно
защищены системные процессы и процессы других пользователей. Перед завершением
всегда запрашивается подтверждение.

**Dev Cleanup — нет**, это намеренно «кувалда»: кнопки бьют по имени всё подряд
(кроме белого списка). Используй осознанно.

## Где лежат данные

`%APPDATA%\WindowsProcessCleaner\` — `config.json` (настройки) и `history.json`
(история). Кнопка «Папка данных» в настройках открывает эту папку.

---

## Права администратора

Приложение всегда запускается от администратора. Это нужно для очистки Standby
Memory через `ntdll!NtSetSystemInformation`. Завершение процессов текущего
пользователя тоже работает.

---

## Технические детали

- **WinAPI:** `CreateToolhelp32Snapshot`, `EnumWindows`, `GetExtendedTcpTable`,
  `OpenProcessToken`/`GetTokenInformation` (владелец процесса),
  `SendMessageTimeout(WM_CLOSE)`, `TerminateProcess` (через `Process.Kill`),
  `NtSetSystemInformation` (Standby Memory), `GlobalMemoryStatusEx`,
  `DwmSetWindowAttribute` (тёмный заголовок).
- **Single-instance** через локальный порт **49876** (обычно свободен): повторный
  запуск просто разворачивает уже открытое окно.
- Мониторинг CPU/простоя обновляется тиком раз в 10 секунд, поэтому «5 минут
  простоя» отсчитываются с момента, как приложение начало наблюдать за процессом.
